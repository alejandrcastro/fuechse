Club Manager

Management system for Füchse Berlin Reinickendorf e.V. · TVBB Berlin

The Club Manager is a two-app web system for managing teams, players, matches, availability and lineups. It runs entirely in the browser — no installation, no build step — backed by Supabase.

🔧 Admin Panel

Full management: import PDFs, configure teams, assign players, generate lineups and enforce blocking rules.

📱 Player App

Mobile-optimized view for players to confirm their availability for upcoming matches.

📄 PDF Import

Auto-imports teams, players and matches from official TVBB Berlin PDFs.

⚖️ TVBB Rules

Stammkader and Stammmannschaft calculated automatically with real-time blocking alerts.

TVBB Rules

Stammkader and Stammmannschaft — exact implementation

📋 Rule 1 — Stammkader

Definition: Players within each category are ranked by LK (best first). The first N form the Stammkader of team 1, the next N of team 2, and so on.

  • N = 6 in Sommer season
  • N = 4 in Winter season
  • A player cannot play in any team with a higher number than their Stammkader team
  • A player can play in teams with a lower number (better teams)

Example (Sommer, 3 Herren teams):

  • Ranking 1–6 → Stammkader Herren 1 (blocked in Herren 2 and 3)
  • Ranking 7–12 → Stammkader Herren 2 (blocked in Herren 3 only)
  • Ranking 13+ → Stammkader Herren 3 (no downward restriction)
📋 Rule 2 — Stammmannschaft

Definition: Playing 2+ matches in team X blocks the player for all teams with a higher number than X, within the same category.

  • Triggered on the 2nd match (1 match → warning ⚠ only)
  • Applies independently per category
  • Does not affect playing in better teams (lower number)
  • Match count can be adjusted manually in the player assignment modal

Example: Player plays 2 matches in Herren II → blocked for Herren III and IV, can still play for Herren I.

Implementation

// getPlayerBlockStatus(playerId, category)

// Rule 1 – Stammkader
const catPlayers = playersInCategory.sort(byLK);
const pos        = catPlayers.indexOf(player);
const skTeam     = catTeams[Math.floor(pos / groupSize)];
const skBlocked  = catTeams.filter(t => t.number > skTeam.number);

// Rule 2 – Stammmannschaft
const highestBlock = catTeams
  .filter(t => matchCounts[player][t] >= 2)
  .sort(byNumber)[0];   // lowest-numbered team with 2+ matches
const smBlocked = catTeams
  .filter(t => t.number > highestBlock?.number);

// Combined: a team is blocked if EITHER rule blocks it

PDF Parsers

Technical documentation

PDF.js text items — fundamentals

PDF.js returns document text as a flat list of items. Each item has item.str (text), item.transform[4] (X), and item.transform[5] (Y from bottom of page).

const x = Math.round(item.transform[4]);
const y = Math.round(viewport.height - item.transform[5]);
// Group into rows by: Math.round(y / 4) * 4
🚫 Critical difference vs pdfplumber/pypdf

PDF.js returns each table cell as a single complete string. The string "Füchse Berlin Reinickendorf e.V. III" arrives as one item — not individual words. Parsers must use item.str.includes(), not word matching.

ResultReport parser

Column layout

The PDF has 4 fixed columns at X ≈ 28, 227, 427, 626 px. Each header row contains one item per column:

{str: "Herren 70 Ostliga Gruppe B",       x: 28 }  // col 0
{str: "Herren Verbandsliga Gruppe C",      x: 227}  // col 1
{str: "Herren Bezirksoberliga I Gruppe D", x: 427}  // col 2
{str: "Herren Bezirksliga I Gruppe B",     x: 626}  // col 3

Algorithm

for each Y-row:
  if row has "Gruppe":
    reset all 4 columns → null
    for each item:
      ci = nearest column index for item.x
      if juvenile (U\d+, Midcourt): active[ci] = null
      elif adult regex matches:     active[ci] = {category, liga}

  elif row has "Reinickendorf":
    for each item containing "Reinickendorf":
      ci  = nearest column index
      col = active[ci]           // skip if null
      roman = suffix in item.str (II/III/IV...)
      register team

Header regex

/^(Herren(?:\s+\d+)?|Damen(?:\s+\d+)?)\s+(.+?)\s+Gruppe\s+[A-Z]$/
//  category (e.g. "Herren 30")         league    group letter

Begegnungen parser

Typical row

Mo. 05.05.2026 10:00 H Verbandsliga Füchse Berlin Reinickendorf e.V. TC Blau-Weiss offen

Steps

  1. Group items by Y (4px tolerance)
  2. Extract date/time: /(?:Mo|Di|Mi|Do|Fr|Sa|So)\.\s+(\d{2})\.(\d{2})\.(\d{4})\s+(\d{2}:\d{2})/
  3. Extract league prefix, map to category via LIGA_CAT_MAP
  4. Home/Away: club first in line → home, club second → away
  5. Team number from roman suffix (II, III, IV...)
  6. Skip juvenile rows (U\d+, Midcourt)
  7. Deduplicate by date+time+category+teamNum+opponent
